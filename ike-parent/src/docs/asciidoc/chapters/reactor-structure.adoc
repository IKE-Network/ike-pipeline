== Reactor Structure

The IKE pipeline is organized as a Maven reactor with a clear separation between the *aggregator* POM (which lists modules) and the *parent* POM chain (which provides inherited configuration).

=== Aggregator vs Parent

Aggregator POM (`pipeline/pom.xml`)::
Lists all modules in dependency order. Running `mvn clean verify` from the project root builds everything. The aggregator is *not* a parent -- no module inherits from it.

Parent POM chain::
`ike-parent` -> `java-parent` -> your project. Each layer adds configuration that child modules inherit automatically.

=== Module Overview

[cols="2,1,4"]
|===
| Module | Packaging | Role

| `minimal-fonts`
| `jar`
| 13 Noto TTF fonts packaged as a Maven artifact. Downloaded once from GitHub releases, cached locally.

| `docbook-xsl`
| `jar`
| DocBook XSL 1.79.2 stylesheets + IKE FO customization layer. Enables the DocBook -> XSL-FO -> XEP pipeline entirely through Maven.

| `koncept-asciidoc-extension`
| `jar`
| AsciidoctorJ SPI extension providing the `k:Name[]` inline macro for formally identified terminology concepts. Renders SVG badges for HTML/PDF and DocBook phrases for XSL-FO. Auto-generates a glossary section with definitions, axioms, and SNOMED CT identifiers.

| `ike-parent`
| `pom`
| Parent POM for all documentation projects. Provides the asciidoctor-maven-plugin configuration, PDF renderer profiles, Kroki diagram integration, and shared extensions.

| `java-parent`
| `pom`
| Extends `ike-parent` with Java compiler, test framework (JUnit 5, AssertJ, Mockito), and JPMS support.

| `example-project`
| `jar`
| Reference implementation. Java source code + AsciiDoc documentation using all pipeline features.
|===

=== Build Order

Maven's reactor sorts modules by their dependency graph. The explicit order in the aggregator POM mirrors this:

. `minimal-fonts` -- no dependencies, built first
. `docbook-xsl` -- no dependencies
. `koncept-asciidoc-extension` -- no dependencies, standalone Java 17 module
. `ike-parent` -- POM packaging, installed to local repo
. `java-parent` -- parent is `ike-parent`
. `example-project` -- parent is `java-parent`, depends on `minimal-fonts`

The `build-all.sh` script exploits this by first installing infrastructure modules (`mvn install -pl '!example-project'`), then building the example project with the selected PDF renderer.
