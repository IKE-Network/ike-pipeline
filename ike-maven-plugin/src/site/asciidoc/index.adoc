= IKE Maven Plugin

IDE-discoverable Maven goals for workspace management,
release automation, and build verification.
Goals appear in IntelliJ's Maven tool window as clickable,
runnable targets with parameter prompts.

== Purpose

Maven plugin goals are discoverable in the IDE.
A developer browsing the Maven tool window sees `ike:init`,
`ike:status`, `ike:feature-finish`, `ike:release` --
clickable targets that integrate workspace and release
operations into the standard Maven workflow.

== Architecture

Release and workspace logic lives in `ReleaseSupport.java`,
a single utility class with clear method boundaries.
Each Mojo delegates to `ReleaseSupport` for version manipulation,
Git operations, and Maven invocations.

== Goals

[cols="2,3",options="header"]
|===
| Goal | Purpose

| `ike:init`
| Create a new IKE Workspace (clone, branch, set version)

| `ike:status`
| Show workspace status (branch, version, uncommitted changes)

| `ike:dashboard`
| Show dashboard of all workspaces

| `ike:feature-finish`
| Strip branch qualifier, merge feature to main, tag

| `ike:ws-checkpoint`
| Create immutable checkpoint from current branch

| `ike:release`
| Transition from SNAPSHOT to release version, build, tag, deploy, bump to next SNAPSHOT

| `ike:ws-release`
| Workspace-level release orchestration — scan dirty components, topo-sort by dependency order, release cascade with automatic cross-reference updates

| `ike:generate-bom`
| Auto-generate a standalone BOM POM from `ike-parent` dependency management (bound to `generate-resources` in `ike-bom`)

| `ike:verify`
| PR validation

| `ike:help`
| Displays available goals and usage
|===

== Design Philosophy

All workspace and release logic lives in the Maven plugin,
providing IDE discoverability, cross-platform compatibility,
and direct integration with the Maven lifecycle.
