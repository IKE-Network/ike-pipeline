== Setting Up a New Project

This chapter walks through creating a new documentation project that inherits the full pipeline.

=== Prerequisites

* *Maven 4+* -- required by the reactor POM (uses `modelVersion 4.1.0`)
* *Java 17+* -- minimum for asciidoctorj; Java 25 if using `java-parent`
* *Node.js 18+* -- optional, only needed for `svgo` (Prawn SVG fixing)

The Maven enforcer plugin validates these at build time with clear error messages.

=== Step 1: Create the POM

Create a minimal `pom.xml` that inherits from the documentation parent (or Java parent if you have source code):

[source,xml]
----
<?xml version="1.0" encoding="UTF-8"?>
<project xmlns="http://maven.apache.org/POM/4.0.0"
         xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
         xsi:schemaLocation="http://maven.apache.org/POM/4.0.0
         http://maven.apache.org/xsd/maven-4.0.0.xsd">
    <modelVersion>4.1.0</modelVersion>

    <parent>
        <groupId>network.ike</groupId>
        <artifactId>java-parent</artifactId>          <!--1-->
        <version>1.1.0-SNAPSHOT</version>
        <relativePath>../java-parent/pom.xml</relativePath>
    </parent>

    <artifactId>my-project</artifactId>
    <version>1.1.0-SNAPSHOT</version>

    <name>My Project</name>

    <dependencies>
        <dependency>
            <groupId>network.ike</groupId>
            <artifactId>minimal-fonts</artifactId>    <!--2-->
        </dependency>
    </dependencies>

    <build>
        <plugins>
            <plugin>
                <groupId>org.asciidoctor</groupId>
                <artifactId>asciidoctor-maven-plugin</artifactId> <!--3-->
            </plugin>
        </plugins>
    </build>
</project>
----
<1> Use `java-parent` for Java projects, `documentation-parent` for docs-only projects
<2> Required for Prawn PDF font rendering; version managed by parent
<3> Declares the plugin so Maven runs it; all configuration inherited from parent

=== Step 2: Create the Index Document

Create `src/docs/asciidoc/index.adoc`:

[source,asciidoc]
----
= My Project Documentation
:author: IKE Community
:email: info@ike-community.org
:revnumber: {project-version}
:revdate: {docdate}
:doctype: book
:toc: left
:toclevels: 3
:sectnums:
:sectnumlevels: 4
:icons: font
:source-highlighter: coderay
:experimental:

== Introduction

Your content here.
----

=== Step 3: Add to the Reactor

Add your module to the aggregator `pom.xml` at the project root:

[source,xml]
----
<modules>
    <module>minimal-fonts</module>
    <module>docbook-xsl</module>
    <module>koncept-asciidoc-extension</module>
    <module>documentation-parent</module>
    <module>java-parent</module>
    <module>example-project</module>
    <module>my-project</module>            <!-- add here -->
</modules>
----

=== Step 4: Build

[source,bash]
----
# HTML only (default profile)
mvn clean verify -pl my-project

# Apache FOP PDF (recommended free renderer, pure Java)
mvn clean verify -pl my-project -Dike.pdf.fop

# WeasyPrint PDF (alternative free, requires Python)
mvn clean verify -pl my-project -Dike.pdf.weasyprint

# Prince XML PDF (commercial, best quality)
mvn clean verify -pl my-project -Dike.pdf.prince

# Prawn PDF (preview quality, no glossary)
mvn clean verify -pl my-project -Dike.pdf.prawn

# All renderers via build-all.sh
./build-all.sh --pdf=fop
----

=== Adding Chapters

Create chapter files in `src/docs/asciidoc/chapters/` and include them from `index.adoc`:

[source,asciidoc]
----
\include::chapters/my-chapter.adoc[]
----

Each chapter file starts at the `==` heading level (the `index.adoc` book structure provides the `=` level).

=== Adding Diagrams

Embed diagrams directly in AsciiDoc. The Kroki server renders them at build time:

[source,asciidoc]
-----
[mermaid]
....
graph LR
    A[Source] --> B[Build] --> C[Output]
....
-----

No local diagram tools needed. Kroki handles Mermaid, PlantUML, GraphViz, and 20+ other diagram types.

=== Cross-Module Includes

To include documentation from another module, add a dependency with the `asciidoc` classifier:

[source,xml]
----
<dependency>
    <groupId>network.ike</groupId>
    <artifactId>other-project</artifactId>
    <version>1.0.0-SNAPSHOT</version>
    <classifier>asciidoc</classifier>
    <type>zip</type>
</dependency>
----

Then include using the `{generated}` attribute:

[source,asciidoc]
----
\include::{generated}/other-project/chapters/shared-chapter.adoc[]
----

The `maven-dependency-plugin` unpacks these to `target/generated-sources/asciidoc/`.
