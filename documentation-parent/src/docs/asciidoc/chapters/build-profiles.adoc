== Build Profiles

All PDF renderers are activated by Maven properties, not by `-P` profile flags. This ensures Maven 4 consumer POM compatibility and allows multiple renderers to be activated simultaneously.

=== Profile Activation

[cols="2,2,2"]
|===
| Profile | Activation Property | Output Directory

| `html`
| Active by default
| `target/generated-docs/html/`

| `pdf` (Prawn)
| `-Dike.pdf.prawn`
| `target/generated-docs/pdf-prawn/`

| `pdf-prince`
| `-Dike.pdf.prince`
| `target/generated-docs/pdf-prince/`

| `pdf-ah`
| `-Dike.pdf.ah`
| `target/generated-docs/pdf-ah/`

| `pdf-weasyprint`
| `-Dike.pdf.weasyprint`
| `target/generated-docs/pdf-weasyprint/`

| `pdf-xep`
| `-Dike.pdf.xep`
| `target/generated-docs/pdf-xep/`

| `all-formats`
| `-Dike.all-formats`
| HTML + Prawn PDF

| `pdf-default`
| `-Dike.pdf.default=RENDERER`
| `target/generated-docs/pdf/` (copy)

| `pom-skip-exec`
| `<packaging>pom</packaging>`
| N/A (skips external renderers)
|===

=== Key Overridable Properties

[cols="3,3,3"]
|===
| Property | Default | Purpose

| `kroki.server.url`
| `https://kroki.komet.sh`
| Kroki diagram rendering server

| `pdf.source.document`
| `index`
| Master document name (without extension)

| `prince.executable`
| `prince`
| Path to Prince XML binary

| `antennahouse.executable`
| `AHFCmd`
| Path to AH Formatter binary

| `weasyprint.executable`
| `weasyprint`
| Path to WeasyPrint binary

| `xep.executable`
| `xep`
| Path to RenderX XEP binary

| `ike.extensions.directory`
| `../documentation-parent/src/extensions`
| Ruby extension directory

| `ike.shared.asciidoc.directory`
| `../documentation-parent/src/shared-asciidoc`
| Shared docinfo directory

| `pdf.print.stylesheet`
| `../documentation-parent/src/theme/ike-print.css`
| CSS Paged Media stylesheet
|===

Override any property on the command line:

[source,bash]
----
mvn verify -Dkroki.server.url=http://localhost:8000 \
           -Dprince.executable=/opt/prince/bin/prince
----

=== `build-all.sh` Behavior

The `build-all.sh` script wraps the reactor build with PDF renderer selection and SVG post-processing:

[source,bash]
----
./build-all.sh                   # HTML + PDF (Prawn default)
./build-all.sh --pdf=prawn       # Prawn with svgo SVG fixing
./build-all.sh --pdf=xep         # DocBook -> XSL-FO -> XEP
./build-all.sh --pdf=prince      # HTML -> Prince XML -> PDF
./build-all.sh --pdf=ah          # HTML -> AH Formatter -> PDF
./build-all.sh --pdf=weasyprint  # HTML -> WeasyPrint -> PDF
./build-all.sh --pdf=none        # HTML only
./build-all.sh --quiet           # Suppress Maven output
----

The script:

. Checks prerequisites (Maven, Java, renderer binaries, Node.js for svgo)
. Installs infrastructure modules: `mvn install -pl '!example-project'`
. Builds the example project with the selected renderer
. For Prawn: runs svgo to inline CSS in SVGs, then rebuilds

=== Build Lifecycle

The following diagram shows when each plugin runs during the Maven lifecycle:

[plantuml]
....
@startuml
|generate-sources|
:maven-dependency-plugin
unpack-doc-dependencies;
note right: Unpack cross-module\nAsciiDoc ZIPs

|generate-resources|
:maven-dependency-plugin
unpack-fonts;
note right: Unpack Noto TTFs\nfrom minimal-fonts

|prepare-package|
:maven-antrun-plugin
prepare-print-assets;
note right: Copy CSS + fonts\nto pdf-html directory

:asciidoctor-maven-plugin
generate print HTML / DocBook;
note right: HTML for Prince/AH/WP\nDocBook for XEP

:exec-maven-plugin
svgo fix SVGs;
note right: foreignObject -> text\n(Prince/AH/XEP only)

|package|
:xml-maven-plugin
DocBook -> XSL-FO;
note right: Saxon-HE + DocBook XSL\n(XEP only)

:exec-maven-plugin
render PDF;
note right: Prince / AH / WP / XEP

:build-helper-maven-plugin
attach-artifact;
note right: Classified PDF artifact\n(prawn/prince/ah/wp/xep)

|verify|
:asciidoctor-maven-plugin
generate HTML + Prawn PDF;
note right: HTML5 backend (always)\nPrawn backend (if activated)

:maven-antrun-plugin
copy-default-pdf;
note right: pdf-RENDERER/ -> pdf/\n(if ike.pdf.default set)

:build-helper-maven-plugin
attach pdf-default;
@enduml
....

NOTE: The `pom-skip-exec` profile sets `ike.pdf.skip.exec=true` for POM-packaging modules. This skips external renderer invocations (Prince, AH, WeasyPrint, XEP, svgo) but does *not* skip the asciidoctor-maven-plugin. POM-packaging modules like `documentation-parent` can still generate HTML and Prawn PDF.
