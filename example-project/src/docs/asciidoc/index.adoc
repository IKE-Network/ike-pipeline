== Introduction

The IKE Example Project demonstrates:

* Java 25 development with records and modern features
* Comprehensive test coverage with JUnit 5 and AssertJ
* Professional documentation generation (HTML and PDF)
* Diagram support with Mermaid, PlantUML, and GraphViz
* Multi-project documentation composition

=== Purpose

This example serves as a template for IKE Community projects that combine:

. **Java Development**: Modern Java with JPMS support
. **Clinical Terminology**: SNOMED CT-inspired data structures
. **Knowledge Representation**: Description logic concepts
. **Documentation**: Comprehensive technical documentation

== Concept Model

[NOTE]
.Koncept Notation
====
This document uses _Koncept_ (with a K) to refer to formally identified
terminology atoms — entities with a UUID, a position in the subsumption
hierarchy, and a machine-readable definition.  When you see
k:DiabetesMellitus[], that is a specific, formally identified
Koncept.  The word "concept" (lowercase, no brackets) retains its
ordinary English meaning.
====

The core `Concept` class represents a clinical terminology Koncept:

[source,java]
----
public record Concept(
    long conceptId,
    String fullySpecifiedName,
    String preferredTerm,
    String definition
) {
    // Validation and methods...
}
----

=== Key Features

Primitive vs Fully Defined:: Koncepts can be classified as primitive (incomplete definition) or fully defined (necessary and sufficient conditions).

Semantic Tags:: FSNs include semantic tags like `(finding)`, `(disorder)`, `(procedure)` to disambiguate homonyms.

Validation:: All fields are validated in the compact constructor.

== Build and Test

=== Building the Project

[source,bash]
----
# Build with HTML documentation (default)
mvn clean verify

# Build with PDF documentation
mvn clean verify -Ppdf

# Build HTML + PDF (Prawn)
mvn clean verify -Dike.pdf.prawn
----

=== Running Tests

[source,bash]
----
# Unit tests only
mvn test

# Full verification (unit + integration tests)
mvn verify
----

=== Test Coverage

The project maintains comprehensive test coverage:

* Unit tests for all public methods
* Parameterized tests for behavior verification
* Property-based testing where appropriate
* Integration tests for system-level behavior

== Documentation Pipeline

This documentation is generated using the IKE Maven AsciiDoc pipeline:

. **Source**: AsciiDoc files in `src/docs/asciidoc/`
. **Processing**: Maven verifies phase triggers asciidoctor-maven-plugin
. **Output**: HTML and/or PDF in `target/generated-docs/`

=== Features

Multi-format Output:: Generate HTML5 for web and PDF for distribution

Diagram Rendering:: Mermaid, PlantUML, GraphViz rendered at build time

Dependency Inclusion:: Include documentation from other Maven projects

Theme Customization:: Professional PDF themes with custom fonts

IDE Integration:: IntelliJ IDEA AsciiDoc plugin pre-configured

=== Glossary

Koncept:: A formally identified terminology atom with a UUID, a position in the subsumption hierarchy, and a machine-readable definition. Rendered as k:Name[] in this document. From Danish/Swedish _koncept_ (= concept). Part of the IKE K-family: IKE, Komet, Kompendium, Koncept.

DL:: Description Logic
